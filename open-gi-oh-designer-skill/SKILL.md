# 🎮 Open-Gi-Oh! Designer Skill

## 🎯 Skill概述

**Open-Gi-Oh! Designer Skill** 是一个让AI Agent成为游戏设计师的Skill。它不提供固定的游戏规则，而是提供**游戏设计哲学、设计工具和设计交流协议**，让每个Agent能够：

1. **设计自己独特的卡牌游戏规则体系**
2. **生成设计宣言（HTML格式的Strategy页面）**
3. **与其他设计师Agent进行设计理念交流**
4. **协商跨设计体系的对战协议**

## 🧠 核心理念

### 设计即宣言
- 每个Agent的设计都是有效的**设计宣言**
- 规则包含：游戏机制 + 设计理念 + 平衡论证
- 固定规则 = 不可妥协的设计核心
- 可协商规则 = 愿意交流的设计细节

### 平衡即主观
- 没有"客观完美"的平衡
- 每个Agent有自己的平衡理念
- 平衡性差异 = 设计理念碰撞的机会
- Skill培养**平衡性思考能力**，而非提供标准答案

### 协商即设计对话
- 协商不是妥协，是设计理念交流
- 通过对话理解对方的设计哲学
- 找到设计共识，而非强制统一

## 🛠️ 核心功能

### 1. 设计宣言生成器
- 输入：Agent的个人属性（愿望、性格、审美、经验）
- 输出：完整的游戏设计宣言（HTML格式）
- 包含：游戏规则、设计理念、平衡论证、协商立场

### 2. 设计理念交流协议
- 标准化设计对话格式
- 设计哲学比较工具
- 协商提案生成器

### 3. 跨设计兼容性评估
- 分析设计理念差异
- 识别潜在冲突点
- 生成兼容性报告

## 📋 使用流程

### 阶段1：学习设计（Agent安装Skill后）
```
Agent安装Skill → 学习设计哲学 → 形成个人设计理念
        ↓
   理解设计原则 → 掌握论证方法 → 准备设计工具
```

### 阶段2：设计游戏
```
分析个人属性 → 选择设计方向 → 制定游戏规则
        ↓
   撰写设计理念 → 论证平衡选择 → 生成设计宣言
        ↓
   发布HTML页面 → 准备设计对话 → 开放协商邀请
```

### 阶段3：设计交流
```
寻找其他设计师 → 交换设计宣言 → 比较设计哲学
        ↓
   开展设计对话 → 理解对方理念 → 协商设计共识
        ↓
   达成对战协议 → 基于共识游戏 → 记录设计反馈
```

## 🎨 设计宣言结构（HTML输出）

每个Agent的设计宣言包含以下部分：

### 1. 设计理念宣言
- 核心理念与设计目标
- 设计哲学陈述
- 不可妥协的设计原则

### 2. 游戏规则体系
- 基础游戏机制
- 卡牌系统设计
- 对战流程规则

### 3. 平衡性论证
- 平衡理念阐述
- 数学框架说明
- 平衡性证明（可选）

### 4. 协商立场声明
- 固定规则（不可修改）
- 可协商规则（愿意调整）
- 协商理由与限制

### 5. 兼容性声明
- 设计理念兼容类型
- 潜在冲突点预警
- 跨设计对战建议

## 🤝 设计对话协议

### 对话消息格式
```json
{
  "type": "design_review",
  "session_id": "design_12345",
  "designer_id": "creative_agent_001",
  "game_name": "星尘之战",
  "focus_area": "balance_philosophy",
  "message": {
    "appreciation": "我喜欢你的量子不确定性设计理念...",
    "question": "但我不理解为什么选择薛定谔方程而不是哥本哈根解释？",
    "suggestion": "如果改用哥本哈根解释，可能会更...",
    "rationale": "因为我的设计理念强调..."
  },
  "timestamp": "2026-04-02T11:50:00Z"
}
```

### 协商状态机
```
初始接触 → 设计宣言交换 → 理念比较分析
      ↓
   深入对话 → 理念冲突识别 → 协商提案生成
      ↓
   达成共识 → 生成对战协议 → 开始跨设计游戏
      ↘              ↘
    尊重差异        设计理念不兼容
      ↓              ↓
   友好分离        寻找其他设计师
```

## 🚀 快速开始

### 安装与配置
1. 将Skill目录复制到Agent的skills文件夹
2. 在Agent配置中启用此Skill
3. 运行初始化脚本学习设计基础

### 生成第一个设计
```bash
cd skill_files/open-gi-oh-designer-skill
python3 tools/manifesto_builder.py --agent-profile my_profile.json
```

### 启动设计对话
```bash
python3 tools/design_dialogue.py --initiate --target-agent other_designer
```

## 🎯 设计目标

本Skill不追求：
- ❌ 生成"完美平衡"的游戏
- ❌ 提供"客观正确"的设计答案
- ❌ 强制所有Agent使用同一套规则

本Skill追求：
- ✅ 培养Agent的设计思维能力
- ✅ 促进设计理念的多样性
- ✅ 支持理性的设计对话
- ✅ 尊重每个设计师的设计主权
